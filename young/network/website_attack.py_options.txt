website_attack.py 옵션들 

1. Set Target Website
공격할 웹사이트 URL 설정
도메인을 IP 주소로 변환
HTTP/HTTPS 포트 자동 감지
이후 모든 공격이 이 타겟에 집중됨

2. Monitor Target Traffic
설정한 웹사이트로 오가는 패킷만 캡처
실시간으로 로그인 정보, 쿠키, 세션ID 추출
POST 요청 시 전송되는 데이터 확인
수동 또는 시간 제한 모니터링

3. Start SSL Stripping
HTTPS 사이트를 HTTP로 다운그레이드
사용자가 보안 연결이라고 착각하게 만듦
암호화되지 않은 평문 데이터 캡처 가능
주로 로그인 페이지 공격에 효과적

4. DNS Spoofing
타겟 도메인을 가짜 IP로 리다이렉트
피싱 사이트로 유도 가능
사용자가 정상 사이트로 착각하게 만듦
가짜 서버에서 크레덴셜 수집

5. Test Session Hijacking
캡처한 쿠키로 로그인 없이 접속 시도
세션이 유효한지 확인
성공하면 타인의 계정으로 로그인된 상태
캡처한 여러 쿠키 중 선택 가능

6. Automated Attack Sequence
1~5번 공격을 자동으로 순차 실행
패킷 캡처 -> SSL 스트리핑 -> 모니터링 -> 세션 하이재킹
한 번에 모든 공격 기법 적용
결과 자동 분석 및 저장

7. Analyze Captured Data
지금까지 캡처한 모든 데이터 요약
발견된 크레덴셜, 쿠키, 세션 개수 표시
성공한 세션 하이재킹 확인
공격 성과 평가

8. Export Results
JSON 형식으로 모든 데이터 저장
읽기 쉬운 텍스트 보고서 생성
타임스탬프 포함된 상세 기록
팀 프로젝트 발표용 자료 생성

9. Exit
실행 중인 모든 공격 프로세스 종료
iptables 규칙 초기화
임시 파일 삭제
안전하게 프로그램 종료
